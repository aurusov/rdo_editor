RESOURCES DESCRIPTION

Resources determine the initial state of the program's (model's) global data base; they are described in a separate object (with extension .rss).

The object of resources has the following format:

$Resources
<resource_description> { <resource_description> }
$End
resource_description
The description of each resource has the following format:
<resource_name> : <resource_type_name> [ <tracing_sign> ]
<parameters_initial_values>
resource_name
Resource name is a simple name. Resources names should be different for each resource, they cannot coincide with the reserved words, nor with resource type names, nor with parameter names.
resource_type_name
Resource type name is a name of one of the resource types described in the object of types.
tracing_sign
When describing resources, tracing sign may be indicated after resource type name (see the description of tracing for details). One of two reserved words is used for determining that sign:
Value Description
trace Means to do the tracing of resources states
no_trace Means not to do the tracing of resources states

By default, no_trace case is accepted, so that reserved word should not compulsorily be printed.

parameters_initial_values
Initial values of resource parameters are determined in positional accordance with the parameters sequence in the type description. The values are indicated with integer or real numerical constant or with a value name according to the parameter type. For parameters with default values specified when describing the type, symbol "*" is allowed instead of their initial values. In that case the parameter will be assigned its default value. If a range of allowed values is determined for a parameter, the accordance is checked up between the initial value and that range.

Remarks

When using chain references, the default value is equal to what is the nearest in the chain. That is why the last parameter of "Resource_2" resource will be assigned the value of "Busy" set for "Parameter_9" parameter and not "Free" set for "Parameter_5"; "Parameter_8" will be assigned "Free".

Examples

$Resources
  Resource_1 : Type_1  0 5.25 100 0.0 Busy 20 10. Free  Loading  Busy
  Resource_2 : Type_1  trace     *  5.25 * 0.0 * 10 10. * * *
  Resource_3 : Type_1  no_trace  *  5.25 * 0.0 * 10 10. * * *
$End
