TRACING OBJECT

RAO simulator creates tracing object (with extension .trc) when there is a descriptor in the run object determining tracing object name. The object has the following format:

Results_file   = <results_object_name>.pmv <creation_date>
Run_file       = <run_name>.smr
Model_name     = <model_name>
Resource_file  = <resources_object_name>.rss
OprIev_file    = <operations_object_name>.opr

$Changes
  <list_of_changes>

$Watching
{ <description_of_an_index_traced> }

$Tracing
  <tracing>

$Status = <run_termination_status> <ending_model_time>
{ <graph_search_statistics> }
DPS_MM <size_of_used_memory>
results_object_name
The results object name is the name of results object wich was created by RAO simulator.
creation_date
Creation date of this tracing object.
run_name
Run name is the name of the run object used in this run.
model_name, resources_object_name, operations_object_name
The names of the model, the resource object and the operation object are determined in the run object.
list_of_changes
A list of changes contains the names and new values of symbol constants, resource parameters and sequence generator bases determined in the run object.
description_of_an_index_traced
Traced indexes are only described if tracing sign trace is indicated in the performance measures description object. The description of a traced index has the following format: 
<index_name> <index_number> <index_kind>
index_name
The index name is one of the names determined in the performance measures description object.
index_number, index_kind
The index number is determined by the order of describing indexes in that object, the index kind is described in that object too.
tracing
Tracing unites information of four kinds: events tracing, resources state tracing, indexes tracing and decision points tracing. Tracing is putted out only if current model time is more then trace start time or less then trace end time. Also the tracing sign must be indicated explicitly in the correcponding object (resource, pattern, index or decision point).
run_termination_status
The status of simulation run termination describes the termination reason, it may be one of the following:
Status Description
NORMAL_TERMINATION The simulation is terminated normally when the termination condition determined for the run is satisfied
NO_MORE_EVENTS There are no more events in the model
USER_BREAK The simulation is terminated because the user has intervened with one of possible commands for stopping RAO simulator
RUN_TIME_ERROR The simulation is terminated because of a run-time error (information about the error is displayed in RAO simulator dialogue window)
graph_search_statistics
Graph search statistics for each decision point of type search have the following format: 
DPS_C  <point_number> <activations_count> <success_count>
DPS_TM <average_search_time> <minimum_search_time>
  <maximum_search_time>
DPS_ME <average_memory_size> <minimum_memory_size>
  <maximum_memory_size>
DPS_CO <average_decision_cost> <minimum_decision_cost>
  <maximum_decision_cost>
DPS_TO <average_opened_tops_count>
  <minimum_opened_tops_count>
  <maximum_opened_tops_count>
DPS_TT <average_tops_count_in_graph>
  <minimum_tops_count_in_graph>
  <maximum_tops_count_in_graph>
DPS_TI <average_tops_count_included_in_graph>
  <minimum_tops_count_included_in_graph>
  <maximum_tops_count_included_in_graph>
DPS_TG <average_count_of_generated_tops>
  <minimum_count_of_generated_tops>
  <maximum_count_of_generated_tops>

These statistics contain decision point number, their activations count and success search terminations count with decision finding. In addition to it these statistics contain average, minimum and maximum values of some search characteristics throughout all search activations. These characteristics are search time (in seconds), memory size used for graph building (in bytes), decision cost, opened tops count, tops count in graph, tops count included in graph (tops, which correspond the same system's state could be included in graph more than once if the new generated top has the smaller path), total generated tops count during the search. These characteristics values for each search activation separately can be found in search termination tracing.

Examples

DPS_C  1  1  1
DPS_TM 193.816  193.816  193.816
DPS_ME 7458816  7458816  7458816
DPS_CO 1.28515e+09  1.28515e+09  1.28515e+09
DPS_TO 10204  10204  10204
DPS_TT 20261  20261  20261
DPS_TI 21109  21109  21109
DPS_TG 38894  38894  38894
