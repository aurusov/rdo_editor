BASIC TERMS

Performing jobs on creation and using SM in RAO medium, user operates with following basic terms:

Model is a set of RAO language objects describing some real object, indexes of the system functioning calculating during simulation run, animation frames and graphical elements used for animation, tracing results.

Run is a single indivisible point of simulation experiment. It is described by the set of objects representing input data and results calculated during simulation run with this input data.

Project is one or more runs united by some common purpose. For example it can be a set of runs aimed at specific object research or performance of one contract for simulation of one or more objects.

Object is a set of information meant for some purposes and making sense for simulation program. Objects composition is caused by RAO method determining the paradigm of CDS representation on RAO language. These objects are examined in detail in following issues. Here we will give only the list of them.

Input data objects are:
  • resource types (with extension .rtp);
  • resources (with extension .rss);
  • patterns (with extension .pat);
  • operations (with extension .opr);
  • decision points (with extension .dpt);
  • constants, functions and sequences (with extension .fun);
  • animation frames (with extension .frm);
  • performance measures (with extension .pmd);
  • run (with extension .smr).

Objects created by RAO simulator during a run:
  • results (with extension .pmv);
  • tracing (with extension .trc).
