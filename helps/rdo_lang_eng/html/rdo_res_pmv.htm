RESULTS OBJECT

RAO simulator creates results object (with extension .pmv) if during a run there is an object describing the indexes (performance measures) required. It has the following format:

Results_file   = <results_object_name>.pmv <creation_date>
Run_file       = <run_name>.smr
Model_name     = <model_name>
Resource_file  = <resources_object_name>.rss
OprIev_file    = <operations_object_name>.opr

$Changes
  <list_of_changes>

$Status = <run_termination_status>
$Result_values <beginning_time_of_statistic_collection>
  <ending_time_of_statistic_collection>
  <calculation_time> <service_information>

  <resulting_statistics>
results_object_name
The name of this results object.
creation_date
Creation date of this results object.
run_name
Run name is the name of the run object used in this run.
model_name, resources_object_name, operations_object_name
The names of the model, the resource object and the operation object are determined in the run object.
list_of_changes
A list of changes contains the names and new values of symbol constants, resource parameters and sequence generator bases determined in the run object.
run_termination_status
The status of simulation run termination describes the termination reason, it may be one of the following:
Status Description
NORMAL_TERMINATION The simulation is terminated normally when the termination condition determined for the run is satisfied
NO_MORE_EVENTS There are no more events in the model
USER_BREAK The simulation is terminated because the user has intervened with one of possible commands for stopping RAO simulator
RUN_TIME_ERROR The simulation is terminated because of a run-time error (information about the error is displayed in RAO simulator dialogue window)
calculation_time
Calculation time is physical duration of the run, measured in seconds.
service_information
Service information contains the data characterising the simulation model's complexity and the computer's speed, including the number of events during the calculations EventCount, the number of checking up the pattern preconditions OperRuleCheckCounter, how many times arithmetic AExpCalcCounter and logical BExpCalcCounter expressions have been calculated, and all those values per one hour of model time and per one second of computer operation time.
resulting_statistics
Statistics on each index is printed in a separate line according to the order the indexes are described in the performance measures object. The line format depends on the index kind.

Formats for all kinds of indexes are given below.


An index of the kind watch_state has the following format:

<index_name> <current_value> <number_of_observations>
<average_value> <sum_of_squares> <minimum_value>
<maximum_value>
index_name
Index name is the name of this index in the performance measures object.
current_value
Current value is the value of logical expression for given index at the moment the results object is put out (TRUE or FALSE).
number_of_observations
The number of observations is how many intervals were there during which the system's state satisfied the conditions (how many times the logical expression changed its value from TRUE to FALSE).
average_value
The average value is the ratio of the period during which the system's state satisfied the condition to the whole period of collecting the index values.
sum_of_squares
The sum of squares is the sum of interval duration squares for intervals during which the system's state satisfied the condition.
minimum_value, maximum_value
The minimum and the maximum values are the minimum and the maximum duration of an interval while the index values were collected.

Examples

Index_1   FALSE  110  0.1099  1.1000  0.1000  0.1000


An index of the kind watch_par has the following format:

<index_name> <current_value> <number_of_observations>
<average_value> <sum_of_squares> <minimum_value>
<maximum_value>

Each observation is a product of interval duration when the parameter's value was constant and the parameter's value.

index_name
Index name is the name of this index in the performance measures object.
current_value
Current value is a parameter's value at the moment the results object is put out.
number_of_observations
The number of observations is how many times the parameter's value changed.
average_value
The average value is calculated as the sum of observations divided by the total length of observations interval, i.e. it is the parameter's mean value during the period.
sum_of_squares
The sum of squares is the sum of observations squares.
minimum_value, maximum_value
The minimum and the maximum values relate to the parameter's values during the observation period.

Examples

Index_3   2  38  1.4507  8615.0102  0  2


An index of the kind watch_quant has the following format:

<index_name> <current_value> <number_of_observations>
<average_value> <sum_of_squares> <minimum_value>
<maximum_value>

Each observation is a product of the interval duration when the quantity of resources did not change and the quantity of resources.

index_name
Index name is the name of this index in the performance measures object.
current_value
The current value is how many resources were there at the moment of putting out the results object.
number_of_observations
The number of observations is how many times the quantity of resources changed.
average_value
The average value is calculated as the sum of observations divided by the observation period duration, i.e. it is the mean quantity of resources for that period.
sum_of_squares
The sum of squares is the sum of observation squares.
minimum_value, maximum_value
The minimum and the maximum values are those for the quantity of resources during the observation period.

Examples

Index_5   9  159  6.2971  4004.3572  1  17


An index of the kind watch_value has the following format:

<index_name> <number_of_observations> <average_value>
<mean_square_deviation> <mean_deviation>
<minimum_value> <maximum_value>

Each observation is the value of arithmetic expression at the moment a temporary resource is eliminated.

index_name
Index name is the name of this index in the performance measures object.
number_of_observations
The number of observations is how many temporary resources were eliminated, their state satisfying the condition.
average_value
The average value is calculated as the sum of observations divided by the number of observations.
mean_square_deviation
The mean square deviation is calculated according to the known formula as square root of the sum of squares of differences between each observation and the mean value, divided by the number of observations diminished by 1.
mean_deviation
The mean deviation is calculated as the mean square deviation divided by square root of the number of observations.
minimum_value, maximum_value
The minimum and the maximum values are those of the arithmetic expression during the observations period.

Examples

Index_7   105  5.5664  2.7038  0.2639  1.6760  13.2216


An index of the kind get_value has the following format:

<index_name> <value>
index_name
Index name is the name of this index in the performance measures object.
value
The value is the arithmetic expression's value at the moment of putting out the object results.

Examples

Index_9  700.6459
