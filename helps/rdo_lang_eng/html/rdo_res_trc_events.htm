EVENTS TRACING

Information about an event is positioned in a separate line with the following format:

E<event_kind> <time> <information_about_the_event>

Event tracing line starts with symbol "E".

event_kind
The second symbol in that line determines the event's kind. The following kinds and corresponding symbols are allowed:
Event kind Description
S Service event
B The start of an action
F The accomplishing of an action
I Irregular event
R Production rule
time
Field "time" contains the moment the event occurs written in real constant format.
information_about_the_event
Then there is information about the event depending on the event's kind:


For service events, that information includes only the service event's number. In the system version considered here, only three service events may take place:

Service event's number Description
1 Starting the tracing
2 Accomplishing the tracing
3 Starting the simulation
4 Accomplishing the graph search in the decision point


For events of starting or finishing an action, the information about such event has the following format:

<action_number> <operation_number> <pattern_number>
<number_of_relevant_resources> <list_of_numbers>
action_number
The action number is internal number that applies within the simulator. That number is necessary for connecting the events of starting and finishing some action, it carries no other useful information.
operation_number, pattern_number
This two fields contain the number of the operation fulfilled during the action and the pattern number for that operation.
number_of_relevant_resources, list_of_numbers
This two fields contain the quantity of the relevant resources and the list of their numbers.


For irregular events, information about an event has the following format:

<irregular_event_number> <pattern_number>
<number_of_relevant_resources> <list_of_numbers>


For production rules, information about an event has the following format:

<action_number> <rule_number> <pattern_number>
<number_of_relevant_resources> <list_of_numbers>
