SYMBOL CONSTANTS, FUNCTIONS AND SEQUENCES DESCRIPTION

To make the program more expressive and convenient to read, symbol constants and functions may be determined in the language.

They are described in a separate object (with extension .fun) that determines symbol constants, functions and sequences. The object's format is as follows:

$Constant
<constant_description> { <constant_description> }
$End
{ <function_or_sequence_description> }

All the constants are described first, then functions and sequences in arbitrary order.

constant_description
The constants are described with the following format:
<constant_name> : <constant_type> = <constant_value>
constant_name
A constant name is a simple name. Names should be different for different constants, they should not coincide with reserved words, resource type names, parameter names or resource names.
constant_type
Constant type is one of the data types allowed in the language. When describing constant types, references are possible on resource parameter types and the types of constants described before.
constant_value
The value of a symbol constant is determined by an integer or real constant, or by a value name according to the constant's type.

Examples

$Constant
  Constant_1 : integer = 10
  Constant_2 : real = 3.14159
  Constant_3 : integer [0..1000] = 0
  Constant_4 : real [0.0..20.0] = 10.0
  Constant_5 : (Free, Busy, Loading, Unloading) = Free
  Constant_6 : such_as Type_1.Parameter_1 = 20
  Constant_7 : such_as Constant_4 = 6.72
  Constant_8 : such_as Type_1.Parameter_5 = Busy
  Constant_9 : (Value_1, Value_2, Value_3,
                 Value_4, Value_5) = Value_1
$End

Remarks

Before a run, symbol constant value may be changed in the run object. If a range of possible value is specified when describing a constant, then it is checked up whether the new value is within that range.

function_or_sequence_description
Functions descriptions and (or) sequences descriptions follow constants descriptions.
