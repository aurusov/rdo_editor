RUN OBJECT

In the run object (with extension .smr), a series of data and modes necessary for controlling the run is indicated. That object consists of two parts. The first part contains the names of the necessary objects and the run modes.

The first lexem in the object must be reserved word Model_name followed by equality sign and the model name. The model name is a simple name; it should not coincide with any name determined before.


Then in the first part of the run object descriptors of the objects and the modes are written in any order. The descriptors have the following format:

<reserved_word> = <value>

Reserved words and possible values are given below with explanations.

Reserved word Description
Resource_file Determines the resource object name. The value is the object's name without its extension.
OprIev_file Determines the name of the operation object. The value is the object's name without extension.
Frame_file Determines the frame object name. The value is the object's name without extension. If the descriptor is not indicated, the frames are not loaded and cannot be displayed.
Statistic_file Determines the resulting performance measures description object. The value is the object's name without extension. If the descriptor is not indicated, the resulting indexes are not collected.
Results_file Determines the results object's name. The value is the object's name without extension. If the descriptor is not indicated, no such object is created.
Trace_file Determines the tracing object name. The value is the object's name without extension. If the descriptor is not indicated, no such object is created.
Show_mode Determines the displaying regime when starting the model. The mode is determined with one of the following reserved words:
  • NoShow - no displaying mode;
  • Monitor - the monitor mode;
  • Animation - state displaying mode (animation).
The default value is NoShow.
Frame_number Determines the frame number among those described in the frame object to be displayed when starting the model if the corresponding mode is ordered. Frame number is an integer constant. The default value is 1.
Show_rate Determines the displaying temporal scale, indicated with real constant equal to the ratio of displaying rate to the model time. For example, with Show_rate = 60.0 the displaying will be 60 times as fast, i.e. one hour of the model time will be displayed during one minute of physical time. The default value is 60.0.
Run_StartTime Determines the model time set when starting the model, indicated with real constant. The default value is 0.0.
Trace_StartTime Determines the moment of starting to put information out into the tracing object, indicated with real constant. The default value is 0.0.
Trace_EndTime Determines the moment of finishing to put information out to the tracing object, indicated with real constant. The default value is 10.0E+30.

It is compulsory to indicate the resource object and the operation object in the first part of the run object. Other descriptors may be absent. In that case, default values apply.


In the second part of the object, breakpoints, run termination condition, new initial values for the constants, resource parameters and sequence generator bases are written in any order.


The run termination condition has the following format:

Terminate_if <logical_expression>

The run will be terminated automatically at the moment the indicated logical expression becomes TRUE.

When there is no condition for terminating the run there is no automatic termination.


The breakpoints apparatus easifies model debugging.

The format for arranging a breakpoint is as follows:

Break_point <breakpoint_name> <logical_expression>

A breakpoint name is a simple name, it should not coincide with any names determined before.

The logical expression tells what condition the system's state has to satisfy. At the moment the indicated logical condition becomes TRUE, the simulation is paused and the relevant message is put out; at that moment RAO simulator shifts to monitor mode.


The format for determining new symbolic constant values is as follows:

<constant_name> = <new_value>

A constant name is one of the constant names described in the constants, functions and sequences object. New value for a symbol constant is determined as an integer or real constant, or as a value name according to the constant type.


The format for new resource parameter values is as follows:

<resource_name>.<parameter_name> = <new_value>

A resource name is one of resource names determined in the resource object. A parameter name is one of parameter names described in the resource types object for the corresponding type. New value is determined either by real/integer constant or by value name according to the parameter type.


Here is the format for determining new sequence generator base value:

<sequence_name>.Seed = <value>

Имя последовательности - это одно из имен, указанных в объекте констант при описании последовательностей. Оно идентифицирует последовательность, для которой задается значение базы. Значение задают численной константой целого типа в диапазоне -2147483647…2147483647.

Sequence name is one of the names indicated in the constants object when describing sequences. That name identifies the sequence the base value is determined for. That value is determined with an integer constant within a range of -2147483648 to 2147483647.

Model_name = example
Resource_file = example
OprIev_file = example
Frame_file = example
Statistic_file = example
Results_file = example
Trace_file = example

             Resource_1.Parameter_1 > 10

Constant_1 = 20

Resource_3.Parameter_5 = Unloading

Sequence_4.Seed = 363252
